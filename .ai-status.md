### Auth & session (Supabase + Resend)

- **Supabase clients:** `src/lib/supabase/client.ts` (browser), `src/lib/supabase/server.ts` (server), `src/lib/supabase/middleware.ts` (session refresh + route checks). Env: `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY` (or `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY`).
- **Middleware** (`middleware.ts`): Protects all `(app)` routes; unauthenticated users are redirected to `/login`. Authenticated users hitting `/login`, `/signup`, or `/forgot-password` are redirected to `/dashboard`. Session is refreshed via `getUser()` and cookies.
- **Server actions** (`src/app/actions/auth.ts`): `signUp` (Supabase Auth + upsert to `public.users` for sync), `logIn`, `logOut`, `resetPassword` (Resend email via Supabase config). All use real Supabase Auth; no mocks.
- **Auth pages:** Login, Sign up, Forgot password use Shadcn Form, Input, Button, react-hook-form, and Zod. Loading: disabled submit + Loader2. Errors: inline destructive text (form-level). Log Out in AppSidebar is wired to `logOut` server action with `useFormStatus` pending state.
- **E2E:** `e2e/auth.spec.ts` covers: unauthenticated `/dashboard` → `/login`; authenticated `/login` → `/dashboard`; sign up → redirect to `/login`; log out → session destroyed. Optional env: `E2E_TEST_USER_EMAIL`, `E2E_TEST_USER_PASSWORD` for login/logout tests.

### Layout architecture (auth vs app)

- **Auth layout** (`app/(auth)/layout.tsx`): Minimalist, centered layout — `min-h-screen flex items-center justify-center bg-background`. Used for `/login`, `/signup`, `/forgot-password`.
- **App layout** (`app/(app)/layout.tsx`): Wraps with `SidebarProvider`, renders `<AppSidebar />` on the left and `<SidebarInset>` as the main content area on the right. No full page reloads when navigating between dashboard, assets, liabilities, transactions, documents, entities, settings.
- **AppSidebar** (`src/components/layout/app-sidebar.tsx`): Shadcn Sidebar primitives, Tailwind semantic tokens (`bg-background`, `text-foreground`, `border-border`), Next.js `<Link>` for client-side routing. Navigation items: Dashboard (PieChart), Assets (TrendingUp), Liabilities (TrendingDown), Transactions (ArrowLeftRight), Documents (FileText), Entities (Building), Settings (Settings). Log Out is wired to `logOut` server action (Supabase Auth). Navigation links update the URL and render the target page without a full browser reload.
